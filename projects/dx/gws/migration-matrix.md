作成日: 2026-03-31

## 凡例
| 記号 | 意味 |
|------|------|
| ✅ | 移行可能（標準的な手順で対応） |
| △ | 部分的に可能（制約あり・アーカイブのみ等） |
| ❌ | 移行不可（再構築・割り切りが必要） |
| — | 該当なし |

## 各移行元の詳細

---

### ① Microsoft 365 / Exchange

#### メール ✅
**方法：** GAMME（Google公式の移行ツール）を使いExchangeサーバーに直接接続
```
Exchange/M365 → GAMME → Gmail
※ 過去メール全量・フォルダ構造ごと移行可能
```
**懸念点：**
- 大量データ（数十GB）は移行に数日かかる
- 共有メールボックス（info@・support@）は別途グループ設定が必要

#### ファイル ✅
**方法：** OneDrive/SharePointから直接Driveへコピー
```
OneDrive → Google Drive for Desktop でコピー
SharePoint → 共有ドライブにフォルダごと移行
```
**懸念点：**
- SharePointの複雑な権限設計をDriveで再現するのに時間がかかる
- SharePointの社内ポータルサイト機能はGoogleサイトで再構築が必要

#### カレンダー ✅
**方法：** GAMMEまたは.icsエクスポートでGoogleカレンダーへ移行
```
Outlookカレンダー → .ics → Googleカレンダーにインポート
```
**懸念点：**
- 繰り返し予定がずれるケースあり（移行後に確認必須）
- 会議室リソースはGWS管理コンソールで再登録が必要

#### 連絡先 ✅
**方法：** OutlookからvCard（.vcf）またはCSVでエクスポート
```
Outlook連絡先 → .vcf/.csv → Googleコンタクトにインポート
```
**懸念点：**
- 個人の連絡先と会社の連絡先が混在している（業務連絡先だけ選別が必要）

#### チャット履歴（Teams） △
**方法：** アーカイブとしてDrive保存のみ
```
Teamsメッセージ → コンプライアンスエクスポートAPI or PDF保存 → Drive
※ Google Chatへの直接移行は不可
```
**懸念点：**
- 過去のやりとりは検索できない状態でのアーカイブになる
- Teams固有の機能（タブ・アプリ連携）はGWSで代替設計が必要

#### ワークフロー（承認フロー） △
**方法：** 履歴はCSV/PDFでアーカイブ。新規はAppSheetで再構築
```
過去の承認記録 → CSV → Drive（参照専用）
新規ワークフロー → AppSheetで再設計
```
**懸念点：**
- 複雑な承認ルートの再構築に2〜4週間かかる

#### ExcelマクロVBA △
**方法：** Google Apps Script（GAS）で書き直し
```
ExcelのVBAマクロ → GASで再実装（1本あたり5〜20万円）
※ シンプルなマクロはSheetsの標準機能で代替できるケースあり
```
**懸念点：**
- 複雑なマクロほど工数・費用が増える
- 移行前に「何本・どんなマクロがあるか」を必ず確認

### ② Bizメール＆ウェブ / さくらメール

#### メール ✅
**方法：** IMAPで接続してGmailに取り込む
```
Bizメール（IMAPサーバー）→ Gmailの「メールをインポート」→ 取り込み
```
**懸念点：**
- POP3設定の場合はサーバーにメールが残っていない可能性あり（ローカルバックアップ確認）
- MXレコード変更後48時間はメール不達のリスクあり

#### ファイル △
**方法：** 元々ファイル管理機能がないため「ないものとして」Driveを新規構築
```
社内サーバーがある場合 → Google Drive for Desktop or rcloneでコピー
社内サーバーがない場合 → Driveをゼロから構築
```
**懸念点：**
- フォルダ構造をゼロから設計する手間がかかる

#### カレンダー —
Bizメールにカレンダー機能は基本なし → Googleカレンダーを新規導入

#### 連絡先 △
**方法：** Outlookなどメールソフトに保存している連絡先をエクスポート
```
Outlookの連絡先 → .vcf/.csv → Googleコンタクト
```
**懸念点：**
- ツール側に連絡先機能がない場合は紙・Excelからの手入力になる

### ③ Yahoo!ビジネスメール

#### メール ✅
**方法：** IMAP設定を有効にしてGmailに取り込む
```
Yahoo!ビジネスメール（IMAP有効化）→ Gmailにインポート
接続先: imap.mail.yahoo.co.jp
```
**懸念点：**
- デフォルトでIMAPが無効になっている場合がある（事前に有効化が必要）
- Yahoo!でドメインを取得している場合、ドメイン移管に1〜2週間かかる

#### ファイル —
Yahoo!ビジネスメールにファイル管理機能なし → Driveを新規導入

#### カレンダー —
該当機能なし → Googleカレンダーを新規導入

#### 連絡先 △
**方法：** Yahoo!の連絡先をCSVエクスポート → Googleコンタクトにインポート
**懸念点：**
- 文字コード（Shift-JIS）の問題で文字化けが起きやすい

### ④ LINE WORKS

#### メール —
LINE WORKSにメール機能はない（一部プランにはあるが使われていないことが多い）
→ GmailをゼロスタートまたはBizメール等から移行

#### ファイル △
**方法：** LINE WORKSのドライブからファイルをダウンロード → Driveにアップロード
```
LINE WORKSドライブ → 手動ダウンロード → Google Driveにアップロード
```
**懸念点：**
- ファイル数が多い場合は手動作業が大変（一括ダウンロードに制限あり）

#### カレンダー △
**方法：** LINE WORKSカレンダーを.icsエクスポート → Googleカレンダーにインポート
**懸念点：**
- 繰り返し予定・共有設定はやり直しが必要

#### 連絡先 △
**方法：** LINE WORKSの連絡先をCSVエクスポート → Googleコンタクトにインポート
**懸念点：**
- LINE WORKSのグループ設定はGWSでは再設定が必要

#### チャット履歴 △
**方法：** 管理コンソールからトーク履歴をCSV/PDFエクスポート → Drive保存
```
LINE WORKSトーク → CSV/PDF出力 → Driveにアーカイブ
※ Google Chatへの直接移行は不可
```
**懸念点：**
- 「LINEじゃないと使いにくい」という現場の抵抗が最大の障壁
- スマホ研修が必須（特に高齢スタッフ）

#### 掲示板・通知 △
**方法：** 内容をコピーしてGoogle ChatのスペースまたはDocsに移行
**懸念点：**
- 構造的な移行は不可。重要なものだけ手動で転記。

### ⑤ Slack ＋ 別メール

#### メール ✅
**方法：** 各メール環境に応じた移行（別メールの種類で対応が変わる）
```
Gmail個人 → 組織Gmailに移行（Takeoutでエクスポート）
Microsoft 365 → GAMMEで移行
Bizメール → IMAPで移行
```

#### ファイル ✅
**方法：** SlackのFile Exportまたは手動でダウンロード → Driveにアップロード
```
Slackファイル → Slack管理画面でエクスポート → Driveに整理
```
**懸念点：**
- Slackに保存されているファイルは整理されていないことが多い
- 有料プランのみファイルの一括エクスポートが可能

#### チャット履歴 △
**方法：** Slack Exportで全チャンネルをJSON形式でダウンロード → Driveにアーカイブ
```
Slack → 管理画面「ワークスペースのデータをエクスポート」→ JSON → Drive
※ Google Chatへのインポートは不可（アーカイブのみ）
```
**懸念点：**
- **無料プランは過去90日分のメッセージしか取得できない**（移行前に確認必須）
- JSONファイルは人間には読みにくいため、重要なやりとりはPDFで別途保存

#### 掲示板・Slackチャンネル △
**方法：** Google ChatのスペースまたはDocuments・Sitesで再設計
**懸念点：**
- Slackの「チャンネルベース」の文化からの移行で抵抗が出やすい
- ピン留め・ブックマークの移行は不可

### ⑥ 個人Gmail（@gmail.com）

#### メール ✅
**方法①：** Google Takeoutで個人Gmailをエクスポート → 組織Gmailにインポート
```
個人Gmail → Google Takeout → Mbox形式 → 組織GmailにIMAPでインポート
```
**方法②：** 個人GmailのIMAPを有効にして組織Gmailから取り込む
```
個人Gmail（IMAP有効化）→ 組織GmailでIMAP接続 → メール取り込み
```
**懸念点：**
- **メールアドレスが変わる**（@gmail.com → @会社名.co.jp）
- 取引先・各種サービスへの新アドレス連絡が必要
- 個人の連絡（家族・友人）と業務メールが混在しているケースあり

#### ファイル ✅
**方法：** 個人Driveのファイルを共有ドライブにコピー
```
個人Googleドライブ → 共有ドライブにコピー（「移動」ではなく「コピー」を使う）
※ 所有権の問題があるため「コピー → 元ファイル削除」の手順が安全
```
**懸念点：**
- 個人のドライブに私物データ（写真・個人書類）も混在していることが多い
- スタッフ自身に「業務ファイルを特定してもらう」作業が必要

#### カレンダー ✅
**方法：** 個人カレンダーを.icsエクスポート → 組織カレンダーにインポート
```
個人Googleカレンダー → 設定 → エクスポート → .ics → 組織カレンダーにインポート
```
**懸念点：**
- 個人の予定（病院・プライベート）も含まれていることがある
- 業務予定だけ選んで移行するか、全量移行するか確認が必要

#### 連絡先 ✅
**方法：** 個人GmailのコンタクトをCSVエクスポート → 組織のコンタクトにインポート
```
Googleコンタクト → エクスポート（CSV/vCard）→ 組織コンタクトにインポート
```
**懸念点：**
- 個人の連絡先と仕事の連絡先が混在（分けてもらう必要あり）

## 共通の注意事項（どの移行元でも起きること）

```
① アドレスが変わる場合は取引先への連絡が必要
② MXレコード変更後24〜48時間はメール不達リスクあり
③ フォルダ・権限設計は移行前に完成させてから作業開始
④ チャット履歴は基本的に「アーカイブ保存」で割り切る
⑤ 重複ファイル・不要データは移行前に整理する
⑥ 並行運用期間（2週間）を必ず設ける
⑦ 各種サービスに登録しているメールアドレスの変更を忘れずに
```
