## 概要

T&Tグループ（拠点17）がGoogle Workspace新規導入・移行を通じて、以下を実現するための要件整理と計画策定を実施。

- 拠点横断の情報共有デジタル化
- 紙中心業務のデジタル化
- セキュリティ・権限管理の確立
- 会議・カレンダー・チャット運用の標準化

**想定プラン:** Business Standard
**ユーザー数:** 24〜27名（提案は30ユーザーで試算）
**コスト試算:** ライセンス54万円/年＋初期40万円＋移行費用 → 総額約100万円

---

## 現状（背景）

| 項目 | 現状 |
|------|------|
| メール | 無料Gmail（個人アカウント） |
| ファイル管理 | ローカルPC・個人マイドライブ（共有ドライブ・NAS未整備） |
| 端末 | 各拠点 Windowsノート2〜3台 |
| コミュニケーション | LINE / LINE WORKS / Microsoft Teams / メール添付が混在 |
| カレンダー | 個人Googleカレンダー（主に本部会議室予約のみ） |
| 書類 | 週報・日報・面談・評価・アンケートが手書き中心 |
| アカウント管理 | 拠点単位で1アカウントを複数人が共有（セキュリティリスク大） |

## 課題一覧

| # | 課題 | 影響 |
|---|------|------|
| 1 | 情報共有がメール添付中心でファイル分散・版管理混乱 | 最新版不明・意思決定遅延 |

| 4 | ファイル資産管理・権限管理不足 | 退職・引継ぎ時に対処困難 |
| 5 | コミュニケーションチャネル乱立 | 重要連絡の抜け漏れ |
| 6 | 会議が対面中心・議事録が属人化 | 移動コスト・記録共有遅延 |
| 7 | 移行計画・セキュリティ設計が未確定 | 切替リスク・補助金整合の懸念 |
| 8 | 個人Gmail移行の作業負荷・コスト管理 | 費用増大リスク |
| 9 | 導入伴走のスコープ不明確 | 追加要望の発生・責任分界が曖昧 |
| 10 | 補助金適用可否の不確実性 | 予算・スケジュール確定が遅れる |

## 導入方針

### ツール役割分担

| 業務 | ツール | 備考 |
|------|--------|------|
| 情報共有・ファイル管理 | Google Drive（共有ドライブ） | 個人マイドライブ原則禁止 |
| 日常連絡（社内） | Google Chat | LINE WORKSは緊急・既読確認用に残す |
| 会議 | Google Meet＋カレンダー | 拠点間オンライン会議標準化 |
| メール | Gmail（組織アカウント） | 対外窓口に特化 |
| 報告書・日報 | Googleフォーム＋Sheets | 自動集約・Looker Studio可視化 |
| リソース管理 | Googleカレンダー（共有リソース） | 会議室・社用車 |

### 移行フェーズ

```
Phase 1: 2週間で主要機能切替（メール・Drive・Chat・カレンダー）
Phase 2: 3拠点PoCで運用確認
Phase 3: 全拠点展開
旧環境: 数ヶ月間バックアップ保持（読み取り専用）
```
