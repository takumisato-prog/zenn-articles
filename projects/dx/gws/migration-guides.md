## 1. Microsoft 365 / Exchange

### よく使う業種・規模
中〜大企業、士業、製造業、医療・福祉

### 移行の難しさ：★★★★☆

### 主なデータ
| データ | 移行先 |
|--------|--------|
| Outlookメール | Gmail |
| OneDrive | Google Drive |
| SharePoint | 共有ドライブ |
| Teamsチャット | Google Chat |
| Excelファイル | Google Sheets（変換 or そのまま保存） |
| カレンダー | Google カレンダー |

### 移行フロー

```
STEP 1: 現状把握（1〜2週間）
├── M365のプラン・ユーザー数確認
├── Exchange サーバーの場合はバージョン確認
├── OneDrive / SharePoint のデータ量
└── Teamsの利用状況（チャンネル・ファイル）

STEP 2: 移行設計（1週間）
├── Driveのフォルダ構造を先に設計
├── ExcelファイルをSheetsに変換するか判断
└── SharePointサイトの構成をDriveに対応させる

STEP 3: メール移行
├── 方法①: GAMME（Google Workspace Migration for Microsoft Exchange）
│   → Microsoftのサーバーに直接接続して移行（最推奨）
├── 方法②: PSTファイルでエクスポート → Gmailにインポート
│   → データ量が少ない場合向き
└── 方法③: IMAPで転送設定

STEP 4: ファイル移行
├── OneDrive → Google Drive for Desktop でコピー
└── SharePoint → 共有ドライブにフォルダごと移行

STEP 5: カレンダー移行
└── Outlookカレンダー → .ics エクスポート → Googleカレンダーにインポート

STEP 6: Teams → Google Chat 移行
├── チャット履歴はそのまま移行不可（アーカイブとして保存）
└── 新規チャットをGoogle Chatに移行

STEP 7: 並行運用（2週間）→ 本番切り替え
```

### 注意点
- **Excelの複雑な関数・マクロ** はSheetsで動かない場合がある（事前検証必須）
- **SharePointの権限設計**が複雑な場合、Drive権限設計に時間がかかる
- TeamsのチャットログはAPIで取得可能だが工数がかかる

### 商談での一言
「OutlookもExcelもそのまま使えます。GWSはOfficeファイルを直接開いて編集できるので、Excelをわざわざ変換する必要もありません。」

### 期間目安：**8〜12週間**

## 2. Bizメール＆ウェブ / さくらメール

### よく使う業種・規模
中小企業全般、老舗企業、個人事業主

### 移行の難しさ：★★☆☆☆

### 主なデータ
| データ | 移行先 |
|--------|--------|
| メール（IMAP/POP3） | Gmail |
| ファイル（なし〜社内サーバー） | Google Drive |

### 移行フロー

```
STEP 1: 現状把握（1週間）
├── メールプロトコル確認（IMAP or POP3）
├── メールデータ量・保存期間
├── ドメイン管理会社の確認（どこで契約しているか）
└── 利用者のメールソフト（Outlook・Thunderbird・Webメール）

STEP 2: ドメインのMXレコード変更（最重要）
├── 現在のMXレコード → Googleのサーバーに向け直す
├── 変更後24〜48時間で切り替わる（DNSの伝播待ち）
└── ドメイン管理会社のコントロールパネルで操作

STEP 3: メール移行
├── IMAPの場合: Gmailの「他のアカウントのメールを確認」でインポート
├── POP3の場合: メールソフトから.emlでエクスポート → Gmailにアップロード
└── 過去メールの移行範囲を決める（全期間 or 直近1年）

STEP 4: 並行運用（1〜2週間）→ 本番切り替え
```

### 注意点
- **MXレコード変更後48時間はメールが届かない可能性がある**（切り替えタイミング要注意）
- POP3の場合はメールがローカルにしか保存されていないことが多い（バックアップ必須）
- ドメイン移管が必要な場合は別途1〜2週間追加

### 商談での一言
「今のドメイン（@〇〇.co.jp）はそのまま使えます。メールアドレスは変わりません。」

### 期間目安：**4〜6週間**

## 3. Yahoo!ビジネスメール

### よく使う業種・規模
小規模企業、個人事業主、EC事業者

### 移行の難しさ：★★☆☆☆

### 主なデータ
| データ | 移行先 |
|--------|--------|
| メール（IMAP対応） | Gmail |
| ファイル（なし） | Google Drive（新規） |

### 移行フロー

```
STEP 1: 現状把握（3〜5日）
├── Yahoo!ビジネスメールのプラン確認
├── ドメインをYahoo!で取得しているか・他社か確認
├── メールデータ量
└── IMAP設定が有効になっているか確認

STEP 2: GWSアカウント作成・ドメイン認証
├── GWSにドメインを登録（MXレコード変更）
└── Yahoo!ビジネスメールのMXレコードを削除

STEP 3: メール移行
├── IMAP経由でGmailに取り込み
│   設定: IMAP → imap.mail.yahoo.co.jp
└── Gmail「設定 → アカウントとインポート → メールをインポート」

STEP 4: 並行運用（1週間）→ 本番切り替え
└── Yahoo!ビジネスメールの解約タイミングを決める
```

### 注意点
- Yahoo!でドメインを取得している場合、**ドメイン移管に1〜2週間かかる**
- Yahoo!ビジネスメールはIMAPに対応しているが、設定が必要
- 解約前に必ずメールの移行完了を確認する

### 商談での一言
「Yahoo!ビジネスメールのメールアドレスはそのまま引き継げます。今のやりとりも全部Gmailに取り込めます。」

### 期間目安：**4〜6週間**

## 4. LINE WORKS

### よく使う業種・規模
飲食・小売・介護・建設・現場系中小企業

### 移行の難しさ：★★★☆☆

### 主なデータ
| データ | 移行先 |
|--------|--------|
| トークメッセージ | Google Chat |
| 共有ファイル | Google Drive |
| カレンダー | Google カレンダー |
| タスク | Google Tasks / Sheets |
| 掲示板 | Google Chat スペース or Docs |

### 移行フロー

```
STEP 1: 現状把握（1週間）
├── 利用機能の洗い出し（トーク・カレンダー・掲示板・タスク・Bot）
├── スタッフのスマホ利用状況（iPhone / Android）
├── LINE WORKSのプラン・契約終了日
└── 「LINEの個人アカウントとの区別ができていないスタッフがいないか」確認

STEP 2: 移行設計
├── Google Chatのグループ構成を設計（部門・現場ごと）
├── LINEの感覚でGoogle Chatを使えるか確認（UIが似ている）
└── トーク履歴を移行するか決める（通常はしない・アーカイブとして保存）

STEP 3: GWSアカウント作成・初期設定
├── 全員のスマホにGmail・Google Chat・Google Driveをインストール
└── スマホ操作研修を先に行う（PCが苦手なスタッフ向け）

STEP 4: Google Chatへの移行
├── 新しいチャットルームをGoogle Chatで作成
├── LINE WORKSと並行して2週間使う
└── LINE WORKSを停止

STEP 5: 定着化
└── 「困ったらここに連絡」の窓口を作る（社内ヘルプデスク）
```

### 注意点
- **スタッフがスマホしか使わない場合が多い** → スマホ操作研修が必須
- LINE WORKSのトーク履歴は**GWSに直接移行できない**（PDF保存 or アーカイブ）
- 「LINEじゃないと使いにくい」という抵抗が出やすい → UIの類似性を強調する

### 商談での一言
「Google Chatはスマホで使えるので、LINE WORKSとほぼ同じ感覚で使えます。画像・ファイルの共有も同じです。」

### 期間目安：**6〜8週間（研修込み）**

## 5. Slack ＋ 別メール

### よく使う業種・規模
IT系・スタートアップ・Web制作・コンサル

### 移行の難しさ：★★★☆☆

### 主なデータ
| データ | 移行先 |
|--------|--------|
| Slackチャンネル・DM | Google Chat |
| Slackファイル | Google Drive |
| メール（各社バラバラ） | Gmail |
| Notionなどのドキュメント | Google Docs / Drive |

### 移行フロー

```
STEP 1: 現状把握（1週間）
├── Slackのプラン確認（無料 / Pro / Business+）
├── チャンネル数・アーカイブの有無
├── Slackと連携しているツール（Zoom・Notion・GitHub等）
├── メールは何を使っているか（Gmail個人 / Microsoft / その他）
└── Slackを完全廃止するか・併用するか決める

STEP 2: 移行設計
├── Slackのチャンネル構成 → Google Chatのスペース構成に変換
├── 連携ツールをGWSで代替できるか確認
│   Zoom → Google Meet
│   Notion → Google Docs / Sites
│   Dropbox → Google Drive
└── Slackの履歴をどこまで移行するか決める

STEP 3: メール統合
├── 各自がバラバラなメールを組織GmailにMXレコード変更で統合
└── Gmailに旧メールの履歴をインポート

STEP 4: Google Chat への移行
├── Slackの無料プランはメッセージ90日制限のため履歴消失注意
├── 有料プランはSlack Export → Drive保存（アーカイブ）
└── 新規コミュニケーションをGoogle Chatに移行

STEP 5: 連携ツールの切り替え
└── Zoom契約 → Google Meet（GWSに含まれるため追加費用ゼロ）

STEP 6: 並行運用（2週間）→ 本番切り替え
```

### 注意点
- **Slackのチャンネル履歴は直接Chatに移行できない**（Exportして保存）
- Slackのフリープランは過去90日しかメッセージが見えない → 移行前に確認
- ITリテラシーが高いスタッフが多いため研修コストは低い

### 商談での一言
「Slackの費用（月2〜15万円）がGWSに含まれるので、ツールを統合しながらコストも下がります。」

### 期間目安：**6〜8週間**

## 6. 個人Gmail（@gmail.com）

### よく使う業種・規模
小規模企業・NPO・福祉・飲食・個人事業主

### 移行の難しさ：★★★☆☆

### 主なデータ
| データ | 移行先 |
|--------|--------|
| 個人Gmailのメール | 組織Gmail |
| 個人Googleドライブ | 共有ドライブ |
| 個人Googleカレンダー | 組織カレンダー |
| 個人Googleフォト | 削除 or 個人で保持 |

### 移行フロー

```
STEP 1: 現状把握（1週間）
├── 何名が個人Gmailを業務利用しているか
├── 個人ドライブに業務ファイルがどのくらいあるか
├── 独自ドメインを持っているか（ない場合は取得が必要）
└── スタッフが「私物と仕事を同じアカウントで使っている」ケースの確認

STEP 2: ドメイン取得（独自ドメインがない場合）
├── 会社名.co.jp or 会社名.com を取得
├── 費用：年間1,000〜3,000円
└── 取得後GWSに登録

STEP 3: 組織アカウント作成
├── 全員に新しいアドレスを発行（例: yamada@会社名.co.jp）
└── admin（管理者）コンソールで全アカウントを一括管理

STEP 4: データ移行（個人 → 組織）
├── メール:
│   個人Gmail → 組織Gmailへ転送設定 or
│   Google Takeout でエクスポート → 組織にインポート
├── ドライブ:
│   個人ドライブのファイルを共有ドライブにコピー
│   （「移動」ではなく「コピー」推奨。所有権の問題を回避）
└── カレンダー:
│   個人カレンダー → .icsエクスポート → 組織カレンダーにインポート

STEP 5: メールアドレス変更の周知
├── 取引先・関係者に新しいアドレスを連絡
├── 個人Gmailに自動返信（「新しいアドレスに変更しました」）
└── 個人GmailからGWSへの転送を一定期間継続

STEP 6: 並行運用（2週間）→ 個人Gmail廃止
```

### 注意点
- **個人ドライブのファイルは「移動」できない**（コピーして共有ドライブに移す）
- スタッフの**私物データ（写真・個人連絡）**と業務データが混在している場合がある
- 「アドレスが変わる」ことへの抵抗 → 取引先への連絡を弊社がテンプレ化してサポート
- 退職者が個人アカウントにデータを持ち去るリスクを解消できる

### 商談での一言
「今の@gmail.comのアドレスは変わりますが、メールもファイルもカレンダーも全部引き継げます。操作感はほぼ同じなので、すぐ使えます。そして一番大事なのは、退職したスタッフのデータが会社に残るようになることです。」

### 期間目安：**6〜8週間**
