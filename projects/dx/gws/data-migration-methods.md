## 1. メール

### 移行方法

| 方法 | 特徴 | 向いているケース |
|------|------|----------------|
| **IMAP転送** | 移行元サーバーにIMAPで接続→Gmailに取り込む | Bizメール・Yahoo!・さくら |
| **GAMME（Google公式ツール）** | MicrosoftのExchangeに直接接続して自動移行 | Microsoft 365 / Exchange |
| **Google Takeout → インポート** | 個人GmailをMbox形式でエクスポート→組織に取り込む | 個人Gmail移行 |
| **PSTエクスポート** | OutlookのデータをPSTファイルに書き出し→インポート | Outlook・Exchange |
| **転送設定** | 旧アドレスに届いたメールを自動で新Gmailへ転送 | 移行期間中の並行運用 |

### 懸念点

```
⚠️ メールアドレスが変わる
   → 取引先への周知が必要。旧アドレスに自動返信設定を忘れずに。

⚠️ 移行できない場合がある（POP3）
   → POP3はサーバーにメールが残らない設定が多い。
     ローカルのメールソフト（Outlook等）にしか残っていない→バックアップ必須。

⚠️ データ量が多いと時間がかかる
   → 10GB以上は数日かかるケースも。移行期間中の業務に影響しない日時を選ぶ。

⚠️ フォルダ構造が崩れることがある
   → IMAP移行後にフォルダ名が文字化けするケースあり。移行後に確認必須。

⚠️ 共有メールボックス（info@・support@）の扱い
   → GWSではグループ設定またはエイリアスで対応。設計が必要。
```

## 2. ファイル・フォルダ

### 移行方法

| 方法 | 特徴 | 向いているケース |
|------|------|----------------|
| **Google Drive for Desktop** | PCにインストール→ドラッグ＆ドロップでコピー | データ量が少ない・担当者が操作できる |
| **rclone** | サーバーから直接Driveへ転送。差分同期・再開可能 | ConoHa・さくら等のVPS・大容量 |
| **弊社代行** | リモートでサーバーに接続し移行作業を全対応 | IT担当者がいない・確実に移行したい |
| **個人Drive → 共有Drive** | 各自がファイルをコピー（移動は所有権の問題あり） | 個人Gmailからの移行 |

### 懸念点

```
⚠️ フォルダ構造を先に決めないと後で混乱する（最重要）
   → 移行前に「誰が何を見られるか」の権限マトリクスを書面で確認・承認してから移行。

⚠️ 「移動」ではなく「コピー」が基本
   → 個人DriveのファイルをそのままShared Driveに移動すると所有権エラーが起きやすい。
     一度コピーしてから元ファイルを削除する手順が安全。

⚠️ 重複ファイルが大量に出てくる
   → 移行前に整理しないとDriveも同じ状態になる。
     「5年以上前のデータ」「同名ファイルが複数ある」ものは事前に整理を依頼。

⚠️ ファイルパスが変わる
   → 社内システム（ExcelのHYPERLINK関数・社内ツールのファイルパス）が壊れる可能性。

⚠️ 容量オーバー
   → Business Standardは1ユーザー2TB。組織全体での上限あり。
     動画・設計図・大量画像がある場合は事前に総量を確認。
```

## 3. カレンダー・スケジュール

### 移行方法

| 方法 | 特徴 | 向いているケース |
|------|------|----------------|
| **.icsエクスポート→インポート** | 標準形式での移行。ほぼ全ツール対応 | サイボウズ・Outlook・LINE WORKS |
| **GWSの移行ツール（GAMME）** | Exchange/M365のカレンダーを自動移行 | Microsoft 365 |
| **手入力** | 件数が少ない場合はシンプル | 重要な予定のみ移行する場合 |

### 懸念点

```
⚠️ 繰り返し予定が崩れることがある
   → 「毎週月曜の会議」「月1回の定例」などの繰り返し設定がずれる場合あり。
     移行後に必ず確認。

⚠️ 共有カレンダーの設定はやり直し
   → 「部門全員が見られる共有カレンダー」は移行後に再設定が必要。

⚠️ 過去の予定まで全部入れるか決める
   → 通常は「今日以降の予定」だけ移行で十分。過去まで移行すると量が増える。

⚠️ リソース（会議室・備品）の設定は別途必要
   → GoogleカレンダーのリソースはGWS管理コンソールで別途登録。
```

## 4. 連絡先・アドレス帳

### 移行方法

| 方法 | 特徴 | 向いているケース |
|------|------|----------------|
| **vCard（.vcf）エクスポート→インポート** | 標準形式。ほぼ全ツール対応 | Outlook・サイボウズ・iPhone |
| **CSV→Googleコンタクトにインポート** | 一括登録が可能 | Excelで管理していた名刺データ |
| **ディレクトリ共有（GWS標準機能）** | 組織全員の連絡先が自動で共有される | GWS導入後は自動 |

### 懸念点

```
⚠️ 個人の連絡先と会社の連絡先が混在している
   → Outlookや個人スマホには私的な連絡先も入っている。
     業務連絡先だけを選んでエクスポートする必要がある。

⚠️ 文字化け（特に旧Outlook）
   → CSV出力時にShift-JISとUTF-8の変換ミスで名前が崩れることがある。
     インポート前にExcelで確認。

⚠️ GWSのディレクトリ情報は別途整備が必要
   → GWS内の組織ディレクトリ（名前・部署・電話番号）はアカウント作成時に手入力。
     既存の名刺データとは別管理になる。
```

## 5. チャット・社内連絡履歴

### 移行方法

| ツール | 移行方法 |
|--------|---------|
| **Slack** | Slack Export（JSON）→ Drive保存（アーカイブ）。Chatへの直接移行は不可 |
| **LINE WORKS** | 管理画面からトーク履歴をCSV/PDFエクスポート → Drive保存 |
| **Microsoft Teams** | コンプライアンスエクスポートAPIで取得 or PDF保存 |
| **個人LINE** | **移行不可**。業務連絡はLINEを使わないようにルール化して対応 |
| **WeChat** | **移行不可**。中国本社連絡専用として残し、国内はGoogle Chatに統一 |

### 懸念点

```
⚠️ チャット履歴はほぼ移行できない
   → Google Chatに過去のSlack・LINE WORKSのメッセージをそのまま持ち込む方法はない。
     「アーカイブとしてDriveに保存する」が現実的な対応。

⚠️ 「履歴が見えなくなる」ことへの抵抗
   → 事前に「過去の履歴はここで見られます」とアーカイブの場所を周知する。

⚠️ Slackの無料プランは90日以前のメッセージが消えている
   → 移行前に確認。必要なやりとりは先にスクリーンショットやPDFで保存。

⚠️ 個人LINEを業務利用している場合は対処が難しい
   → 「個人の端末の個人アカウント」は強制的に消せない。
     Google Chatへの移行をルール化して、段階的に切り替えていく。
```

## 6. ワークフロー・申請履歴

### 移行方法

| 目的 | 移行方法 |
|------|---------|
| **過去の申請・承認記録を参照したい** | CSVエクスポート → Sheets変換 or PDF → Drive保存 |
| **現在進行中の申請を止めたくない** | 並行運用期間（2週間）を設けて切り替え |
| **同じワークフローをGWSで再現したい** | AppSheetで再構築（2〜4週間） |

### 懸念点

```
⚠️ ワークフローのデータは直接移行できない
   → サイボウズ・kintoneのワークフローをそのままGWSに持ち込む方法はない。
     「CSV/PDFでアーカイブ保存」＋「AppSheetで再構築」が標準対応。

⚠️ AppSheetの再構築に工数がかかる
   → 単純な1段階承認: 1〜2週間
     複雑な多段階承認・条件分岐: 4〜8週間
     費用: 15〜50万円程度

⚠️ 移行期間中に申請業務が滞るリスク
   → 新旧システムを並行運用する期間を必ず設ける。
     「この日から新システム」という切り替え日を全員に周知。

⚠️ 承認ルートを再設計する必要がある
   → 現状の承認ルートが「属人的・口頭ベース」の場合、
     AppSheet構築前に「誰が誰に承認するか」を文書化してもらう必要がある。
```

## 7. Excelファイル

### 移行方法

| 方法 | 特徴 |
|------|------|
| **Driveにアップロードしてそのまま使う** | .xlsxのままGoogleドライブで開ける。Excelとの互換性を保てる |
| **Google Sheetsに変換** | GWSネイティブ形式。Geminiや自動化との連携が便利 |
| **用途によって使い分け** | 社外共有・精密な印刷が必要→Excel / 社内共有・自動化→Sheets |

### 懸念点

```
⚠️ 複雑なマクロ（VBA）はSheetsで動かない
   → ExcelのVBAマクロはGoogle Apps Script（GAS）で書き直しが必要。
     工数: 1本あたり5〜20万円。事前に「何本のマクロがあるか」を確認。

⚠️ 特定のExcel関数がSheetsで対応していない
   → ほとんどは対応しているが、一部の関数（GETPIVOTDATA等）は動作が異なる。
     重要なファイルは事前に動作確認が必要。

⚠️ 印刷レイアウトが崩れることがある
   → 細かい印刷設定（ページ設定・余白）はSheets変換後に崩れやすい。
     請求書・納品書など印刷前提のファイルは変換しないことを推奨。

⚠️ 「Excelで送って」という取引先への対応
   → Drive上で.xlsxとして保存していれば、Excelファイルとしてダウンロード・送付できる。
     取引先対応は問題なし。
```

## 8. 動画・大容量ファイル

### 移行方法

| 方法 | 特徴 |
|------|------|
| **rclone** | 大容量向け。サーバーから直接Driveに転送。差分同期可能 |
| **Google Drive for Desktop** | PCを経由してアップロード。データ量が少ない場合向き |
| **弊社代行** | リモートで作業代行。IT担当不在の場合に最適 |

### 懸念点

```
⚠️ アップロードに時間がかかる
   → 100GBのデータで光回線でも数時間〜1日かかる。
     深夜や週末に実行するよう計画する。

⚠️ Google Driveの動画はYouTubeのように直接再生できる
   → 移行後は専用の動画プレイヤー不要になるが、再エンコードの時間がかかる。

⚠️ Business Standardの容量上限
   → 1ユーザーあたり2TB。ただし組織全体のプール制なので実質的にはかなり大きい。
     動画・設計図・医療画像など大容量データがある業種は事前に総量を確認。

⚠️ 外部共有の設定に注意
   → Driveに入れたファイルは設定次第で社外に共有できてしまう。
     機密性の高い動画（会議録画・製造映像等）は共有制限設定を必ず行う。
```

## 9. 顧客データ・取引先リスト

### 移行方法

| 方法 | 特徴 |
|------|------|
| **CSV → Google Sheets** | 既存のExcel顧客リストをSheetsに変換 |
| **CSV → Googleコンタクト** | 取引先の連絡先をGoogleコンタクトに一括登録 |
| **既存CRMとの連携** | Salesforce・HubSpot等はGWSとAPI連携が可能 |

### 懸念点

```
⚠️ 個人情報保護法への対応が必要
   → 顧客データをDriveに入れる場合、外部共有制限・アクセスログ設定が必須。
     特に医療・福祉・士業は慎重に設計。

⚠️ 文字コードの問題
   → 古いシステムからCSVエクスポートすると文字化けが起きやすい。
     Excelで一度開いて確認してからSheetsにインポートする。

⚠️ 重複データの整理
   → 長年使っているリストには同一取引先が複数登録されていることが多い。
     移行前に整理をお願いする（弊社で代行も可）。
```

## 10. パスワード・認証情報

### 移行方法

| 方法 | 特徴 |
|------|------|
| **パスワードマネージャー（1Password等）** | GWSとは別管理。移行後も継続使用 |
| **Google パスワードマネージャー** | Chromeに統合。GWSアカウントでSSO管理 |
| **二段階認証の再設定** | GWS移行後は全員の二段階認証を再設定 |

### 懸念点

```
⚠️ 各サービスのパスワード（旧メールアドレス紐付け）のリセットが必要
   → メールアドレスが変わると、各種サービス（EC・銀行・ツール等）に
     登録しているメールアドレスを全て変更する必要がある。
     「どのサービスに旧アドレスを登録しているか」のリスト化を依頼。

⚠️ 二段階認証のバックアップコードを保存しておく
   → GWSの二段階認証はスマホのSMS/認証アプリで設定。
     スマホを紛失したときのためにバックアップコードを印刷して保管。

⚠️ 旧アカウントの解約タイミング
   → メールアドレスが変わるサービスは全部変更し終わってから旧アドレスを解約する。
     解約後に「パスワードリセットメールを送ってください」ができなくなる。
```

## データ別 移行の優先順位と難易度まとめ

| データ | 移行可否 | 難易度 | 優先度 |
|--------|---------|--------|--------|
| メール | ✅ ほぼ全量移行可 | ★★★☆☆ | 最優先 |
| ファイル | ✅ 移行可 | ★★☆☆☆ | 最優先 |
| カレンダー | ✅ 移行可 | ★★☆☆☆ | 高 |
| 連絡先 | ✅ 移行可 | ★☆☆☆☆ | 中 |
| チャット履歴 | △ アーカイブのみ | ★★★★☆ | 低（割り切り） |
| ワークフロー履歴 | △ アーカイブのみ | ★★★☆☆ | 中（要確認） |
| Excelマクロ | △ 再構築が必要 | ★★★★★ | 要事前調査 |
| 動画・大容量 | ✅ 移行可（時間かかる） | ★★☆☆☆ | 低 |
| 顧客データ | ✅ CSV変換で移行可 | ★★☆☆☆ | 高（セキュリティ注意） |
| パスワード | ⚠️ 各サービス個別対応 | ★★★★☆ | 高（見落としやすい） |
