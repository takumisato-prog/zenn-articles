# AIツールで静岡の地域情報発信を自動化した話——この記事を書いた仕組みを公開

少し変わった記事を書きます。

この静岡DEEP連載は、私（人間のライター）が一人で書いているように見えますが、実際にはAIと自動化ツールが深く関わっています。記事の構成、下書きの生成、note への投稿、画像の作成——これらの大部分が半自動化されています。

「地域情報発信を続けること」の最大の壁は、継続できないことです。週1本の記事を1年間書き続けるには52本の記事が必要で、それを一人でこなすのは体力的にも時間的にも厳しい。そこでシステムを作りました。今日はその仕組みをまるごと公開します。

## 全体の構成図

```
テーマ・アイデア → Claude API → 下書き生成
                                    ↓
                            人間によるレビュー・修正（30〜60分）
                                    ↓
                  Playwright自動化 → note投稿
                                    ↓
                   Google Drive → 記事アーカイブ保存
                                    ↓
                       cron（定期実行）→ 週次で自動スケジュール
```

5つのコンポーネントが連携して動いています。順番に説明します。

## コンポーネント1——Claude API（記事生成の心臓部）

記事の下書き生成にはAnthropicのClaude APIを使っています。使用モデルは`claude-sonnet-4-6`。

プロンプトはかなり細かく設計しています。単に「静岡について記事を書いて」とやっても、月並みな観光ガイドしか出てきません。実際のシステムプロンプトは以下のような構成です：

```
あなたは静岡の深い魅力を発信するライターです。
以下のルールを必ず守ってください：

- 具体的な数字・固有名詞・エピソードを必ず含める
- 「静岡は自然が豊か」等の月並みな表現は使わない
- 読者に語りかける文体（です・ます調）
- 2000〜2500文字
- H2・H3の見出しを適切に使う
- 最後に「次回は〜」の予告を入れる

[記事テーマ]
タイトル: {title}
シリーズ: {series}
補足情報: {context}
```

このプロンプトに対してAPIを呼び出すNode.jsスクリプトが動きます。

```javascript
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic();

async function generateArticle(title, series, context) {
  const message = await client.messages.create({
    model: "claude-sonnet-4-6",
    max_tokens: 4096,
    system: SYSTEM_PROMPT,
    messages: [
      {
        role: "user",
        content: `タイトル: ${title}\nシリーズ: ${series}\n補足: ${context}`
      }
    ]
  });

  return message.content[0].text;
}
```

1本の記事生成にかかる時間は約30〜90秒、API費用は1本あたり約15〜25円です（2024年時点のSonnetモデル料金）。

## コンポーネント2——人間のレビュー（省略できない工程）

AIが生成した下書きを、そのままnoteに投稿することはしていません。必ず人間がレビューして修正する工程を入れています。

AIの下書きには得意・不得意があります。

**得意なこと**：
- 全体の構成を整える
- 見出しのバランスを取る
- 一般的な情報を正確に組み合わせる
- 読みやすい文章のリズムを作る

**苦手なこと・よくある問題**：
- 地元民しか知らないディテールの正確さ
- 「本当にそうだっけ？」という数字や固有名詞の検証
- 地元特有のニュアンス（「さわやか」への静岡人の感情など）
- 冒頭の導入文が型にはまりがちになる

レビューにかかる時間は記事によって異なりますが、平均40分程度。修正量は軽いもの（文体調整・数字確認）だけのときもあれば、1セクション全体を書き直すこともあります。

AIは「8割の精度で下書きを出す機械」、人間は「その8割を10割に仕上げる編集者」という役割分担です。

## コンポーネント3——Playwright（note投稿の自動化）

レビュー済みの記事をnoteに投稿する作業も自動化しています。Playwrightというブラウザ自動化ライブラリを使います。

```javascript
import { chromium } from 'playwright';

async function postToNote(articleContent, title, tags) {
  const browser = await chromium.launch({ headless: false });
  const page = await browser.newPage();

  // noteにログイン
  await page.goto('https://note.com/login');
  await page.fill('#email', process.env.NOTE_EMAIL);
  await page.fill('#password', process.env.NOTE_PASSWORD);
  await page.click('button[type="submit"]');

  // 新規記事作成ページへ
  await page.goto('https://note.com/new');

  // タイトルと本文を入力
  await page.fill('[data-placeholder="タイトル"]', title);
  await page.fill('.ProseMirror', articleContent);

  // タグ設定・公開
  await setTags(page, tags);
  await page.click('button:has-text("公開する")');
}
```

これにより、記事の投稿作業（コピペ・フォーマット確認・タグ設定・公開）にかかる時間が、1本あたり約15分から2〜3分に短縮されました。

ただしnoteのUIアップデートで定期的にセレクタが変わり、スクリプトが壊れることがあります。Playwright自動化の維持コストとして「月1〜2回のスクリプト確認・修正」が必要です。

## コンポーネント4——Google Drive（アーカイブとコンテンツ管理）

全記事のマスターデータはGoogle Driveのスプレッドシートで管理しています。

| 列 | 内容 |
|---|---|
| ID | 記事番号 |
| タイトル | 記事タイトル |
| シリーズ | A/B/C/D |
| ステータス | 企画中/執筆中/レビュー済/投稿済 |
| note URL | 投稿後のURL |
| 投稿日 | 実際の投稿日時 |
| PV数 | 週次で手動更新 |
| スキ数 | 週次で手動更新 |

Google Apps Script（GAS）でスプレッドシートとSlackを連携させており、「投稿済み」ステータスになったタイミングでSlackの特定チャンネルに通知が来ます。「記事○○を投稿しました」という自動通知で、投稿漏れを防いでいます。

記事の本文ファイルはGoogle Driveのフォルダにmarkdown形式で保存。同時にGitHubリポジトリにも同期させており、バージョン管理もできています。

## コンポーネント5——cron（定期実行とスケジュール管理）

毎週の記事投稿スケジュールをcronで自動管理しています。

具体的には毎週月曜朝9時に実行されるスクリプトが：

1. スプレッドシートから「レビュー済」の記事を1本取得
2. その週の投稿予定日・時間を自動で設定
3. 担当者（自分）にSlack通知を送る

```bash
# crontab設定
0 9 * * 1 /usr/local/bin/node /path/to/schedule-weekly.js
```

noteの予約投稿機能（有料プラン）と組み合わせることで、「月曜に確認→水曜18時に自動投稿」という流れが自動化されています。

## システム全体のコストと効果

月次のコストを計算すると：

- Claude API：月4〜8本の記事生成で約500〜1,000円
- noteプレミアム（予約投稿機能）：月500円
- VPS（cron実行サーバー）：月700円
- その他ツール：月500円程度

合計：月2,200〜2,700円

これで週1本の記事更新が維持できています。純粋な執筆時間は1本あたり40〜60分（レビュー時間）で、以前のフルマニュアル執筆（1本あたり3〜4時間）から大幅に削減されました。

## 「AI発信」の倫理的な問題について

最後に正直な話をします。

「AIが書いた記事を人間が書いたように見せることは問題ではないか」という指摘は当然あります。

私のスタンスは「AIは道具、判断と責任は人間」というものです。この記事に書いた内容の正確性・倫理性・地元への敬意——これらは人間のレビュー工程で担保しています。AIが出力したものをそのまま放流することはしていません。

むしろ「AIを使っていることを隠す」方が問題だと思っているので、このメタ記事を書きました。地域情報発信の自動化という仕組みを公開することで、同じような課題（続かない、時間がない、一人でやっている）を持つ地域メディアの参考になれば、それは価値があると考えています。

静岡DEEPは、AIを使いながらも、静岡という場所への本物の関心と取材から成り立っています。それだけは断言できます。
