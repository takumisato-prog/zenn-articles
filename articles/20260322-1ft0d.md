---

**【1/10】そもそも「スキルファイル」って何？**

Claude Codeには、AIの振る舞いを定義できる「スキルファイル」という仕組みがある。

マークダウンで書いた指示書を読み込ませるだけで、汎用AIが「専門家AI」に変わる。

```markdown
# あなたはSEOの専門家です

## 役割
- キーワード選定と競合分析
- 検索意図に基づいたコンテンツ設計
- メタタグ・構造化データの最適化

## 回答スタイル
- 必ず根拠となるデータを示す
- 施策の優先順位を明記する
```

たったこれだけで、SEO専門家AIができあがる。

**【2/10】汎用AIと専門家AIの違い、実験してみた**

「記事のSEO改善点を教えて」と聞いた場合の違い↓

**汎用AI**:「タイトルにキーワードを入れ、見出しを整理し、内部リンクを増やしましょう」

**SEO専門家AI（スキルファイルあり）**:「検索ボリューム2,400/月の"Claude Code 使い方"をH1に入れ、LSIキーワードとして"プロンプト"と"エージェント"をH2に配置。競合上位3記事の平均文字数は3,800字なので現状の2,100字から増量が必要です」

具体性がまるで違う。

**【4/10】スキルファイルの基本構成、テンプレート公開**

どの専門家AIにも共通して入れてる構成がこれ↓

```markdown
# [専門家名]

## 役割定義
（何者で、何を担当するか）

## 専門知識の範囲
（得意領域と対応スコープ）

## 回答のルール
（フォーマット・根拠の示し方・優先順位の付け方）

## 禁止事項
（専門外への踏み込み禁止など）

## 連携先
（他の専門家に引き継ぐべき条件）
```

この5項目を埋めるだけで、ちゃんと機能する専門家AIができる。

**【5/10】一番苦労した「法務レビュアーAI」の作り方**

法務AIは、間違えると実害が出るので設計が一番難しかった。

ポイントは「禁止事項」を徹底的に書くこと。

```markdown
## 禁止事項
- 「この契約は問題ありません」と断言しない
- 日本法以外の法律を根拠に判断しない
- 税務・会計領域の判断をしない
- 回答の末尾に必ず「専門家への最終確認を推奨」と明記する

## 回答スタイル
- リスクを必ず「高・中・低」で評価する
- 問題のある条文は引用した上で指摘する
- 修正例を必ず併記する
```

「できること」より「できないこと」を明示するのが、信頼できる専門家AIのコツ。

**【6/10】専門家AIを「チームで連携」させる仕組み**

10体を個別に動かすだけじゃもったいない。自分は「PMエージェント」を別に作って、タスクを振り分けさせてる。

```markdown
# PMエージェント

## 役割
ユーザーの依頼を受け取り、適切な専門家に
タスクを割り振る統括マネージャー

## 振り分けルール
- SEO関連 → SEOストラテジスト
- 文章作成 → コピーライター
- 数値分析 → データアナリスト
- 契約確認 → 法務レビュアー
- 複合案件 → 複数の専門家に並列依頼
```

「新規サービスのLP作りたい」と投げると、PMが自動でコピーライター・UXリサーチャー・SEOストラテジストに分解して渡してくれる。

**【7/10】実際の運用で気づいた「スキルファイル3つの落とし穴」**

やってみてハマったポイントを正直に書く。

**落とし穴①: 役割が広すぎる**
「マーケター」じゃなく「SNS広告運用特化」くらい絞らないとぼんやりした回答になる。

**落とし穴②: 禁止事項が少ない**
何でも答えようとして専門性が薄まる。「〇〇はしない」を書くほど精度が上がる。

**落とし穴③: 更新しない**
最初に作って満足しがち。月1回は回答品質をチェックして、スキルファイルを改訂する習慣が大事。
